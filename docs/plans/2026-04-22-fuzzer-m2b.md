# Fuzzer M2b Implementation Plan — Generic Tx Shape + Core Tx Types

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Expand the fuzzer's tx-type coverage by making the tx representation generic (`Fields map[string]any`) so new tx types can be added without touching the runner, then add three core tx types (AccountSet, SetRegularKey, TicketCreate) via the new shape.

**Architecture:** Replace `generator.Tx`'s hard-coded field set with a `Fields` map and a `Secret` string. Add `rpcclient.SubmitTxJSON(secret, txjson)` as the generic submission path. Collapse `runners.submitTx`'s per-type switch into a one-liner that calls `SubmitTxJSON`. Three new builders exercise the new plumbing: AccountSet (flags), SetRegularKey (optional field), TicketCreate (simple numeric field).

**Tech Stack:** Go 1.24, existing `rpcclient` / `generator` / `runners` packages; no new external deps.

**Branch:** `fuzz/m2b` stacked on `fuzz/m2a` (#9).

**Scope:**
- In: `Tx` refactor, `SubmitTxJSON`, runner dispatch simplification, AccountSet / SetRegularKey / TicketCreate builders.
- Out (deferred): OfferCancel (needs in-memory offer tracking), amendment-gated families (Escrow, Check, AMM, NFT — each is its own mini-milestone), state-prerequisite tracking beyond the dense mesh from M2a.

```go
// SubmitTxJSON submits an arbitrary tx_json object under the given secret.
// Generic path used by fuzzer tx types that don't map to SubmitPayment/Trust/
// Offer/PaymentIOU. The tx_json must include TransactionType and Account.
// fee_mult_max is not added here — callers include it in tx_json if they need it,
// or rely on the wrapping Call to attach it (current client.go behavior).
func (c *Client) SubmitTxJSON(secret string, txJSON map[string]any) (*SubmitResult, error) {
	raw, err := c.Call("submit", map[string]interface{}{
		"secret":  secret,
		"tx_json": txJSON,
	})
	if err != nil {
		return nil, err
	}
	return parseSubmitResult(raw)
}
```

Run tests; expect PASS.

#### Step 2: Refactor `generator.Tx` to Fields map

In `sidecar/internal/fuzz/generator/generator.go`, replace the `Tx` struct with:

```go
// Tx is the fuzzer's transport-neutral transaction representation. Fields
// holds the full tx_json object (TransactionType + all tx-specific fields);
// Secret is kept separate so it's never accidentally serialized on-wire.
type Tx struct {
	Fields map[string]any `json:"-"`
	Secret string         `json:"-"`
}

// TransactionType returns the TransactionType field, for convenience.
func (t *Tx) TransactionType() string {
	if s, ok := t.Fields["TransactionType"].(string); ok {
		return s
	}
	return ""
}
```

```go
package generator

import (
	mathrand "math/rand/v2"
)

// asfFlags is the set of asf* flags a well-formed AccountSet can toggle.
// Source: rippled SField list (asfRequireDest=1, asfRequireAuth=2,
// asfDisallowXRP=3, asfDisableMaster=4, asfAccountTxnID=5, asfNoFreeze=6,
// asfGlobalFreeze=7, asfDefaultRipple=8, asfDepositAuth=9). Flag 4
// (DisableMaster) is excluded — enabling it can permanently blackhole
// the account if no regular key is set.
var asfFlags = []uint32{1, 2, 3, 5, 6, 7, 8, 9}

// AccountSet generates a well-formed AccountSet tx that toggles a random
// asf* flag. Half the time sets a flag, half the time clears one.
func (g *Generator) AccountSet(r *mathrand.Rand) (*Tx, error) {
	acct := g.pool.Pick(r)
	flag := asfFlags[r.IntN(len(asfFlags))]
	fields := map[string]any{
		"TransactionType": "AccountSet",
		"Account":         acct.ClassicAddress,
	}
	if r.IntN(2) == 0 {
		fields["SetFlag"] = flag
	} else {
		fields["ClearFlag"] = flag
	}
	return &Tx{Fields: fields, Secret: acct.Seed}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "AccountSet",
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.AccountSet(r) },
	})
}
```

```go
package generator

import (
	mathrand "math/rand/v2"
)

// SetRegularKey randomly either (a) sets the account's RegularKey to another
// pool account's address, or (b) clears it (omits the RegularKey field).
// Note: in production, clearing a regular key while the master key is disabled
// (asfDisableMaster) would blackhole the account. AccountSet generator excludes
// asfDisableMaster for exactly this reason.
func (g *Generator) SetRegularKey(r *mathrand.Rand) (*Tx, error) {
	acct := g.pool.Pick(r)
	fields := map[string]any{
		"TransactionType": "SetRegularKey",
		"Account":         acct.ClassicAddress,
	}
	if r.IntN(2) == 0 {
		other := g.pool.Pick(r)
		for other.ClassicAddress == acct.ClassicAddress {
			other = g.pool.Pick(r)
		}
		fields["RegularKey"] = other.ClassicAddress
	}
	// else: clear by omitting RegularKey (XRPL semantic: absent = clear).
	return &Tx{Fields: fields, Secret: acct.Seed}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "SetRegularKey",
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.SetRegularKey(r) },
	})
}
```

```go
package generator

import (
	mathrand "math/rand/v2"
)

// TicketCreate reserves 1..5 tickets for the sender. Low numbers keep reserve
// pressure bounded — each ticket counts against the owner-count reserve until
// consumed. Max is 250 per tx (XRPL hard cap); we stay well under to avoid
// reserve-threshold flapping during the fuzz.
func (g *Generator) TicketCreate(r *mathrand.Rand) (*Tx, error) {
	acct := g.pool.Pick(r)
	count := uint32(r.IntN(5) + 1)
	return &Tx{
		Fields: map[string]any{
			"TransactionType": "TicketCreate",
			"Account":         acct.ClassicAddress,
			"TicketCount":     count,
		},
		Secret: acct.Seed,
	}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "TicketCreate",
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.TicketCreate(r) },
	})
}
```
