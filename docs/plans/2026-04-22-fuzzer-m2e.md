# Fuzzer M2e Implementation Plan — Escrow Family + State Tracker

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add the first amendment-gated tx family (Escrow: Create / Finish / Cancel) and introduce the runtime state-tracker infrastructure that Escrow Finish/Cancel — and every future stateful tx family — needs. The runner feeds the tracker with successful EscrowCreate submits; the generator's selector skips Finish/Cancel when no tracked escrows exist.

**Architecture:**
- `rpcclient.SubmitResult` gains a `Sequence uint32` parsed from `tx_json.Sequence` in the submit response — needed so the runner can record `(owner, OfferSequence)` for each EscrowCreate.
- New `generator.Tracker` struct holds per-family state. First entry: `EscrowTracker` with `Record(owner, seq)` / `PickOpen(r)` / `Count()`. Attached to `Generator` alongside `pool` and `mutator`.
- `generator.CandidateTx` gains an optional `CanBuild func(g *Generator) bool` — `EscrowFinish` / `EscrowCancel` use it to gate themselves on `g.Tracker().Escrows().Count() > 0`.
- Runner's tx loop: on successful EscrowCreate submit, extract Sequence + call `tracker.RecordEscrow(...)`.

**Tech Stack:** Go 1.24, existing generator / rpcclient / runners packages.

**Branch:** `fuzz/m2e` stacked on `fuzz/m2c` (#12).

**Scope:**
- In: `SubmitResult.Sequence`; `Tracker` + `EscrowTracker`; `CandidateTx.CanBuild`; Escrow Create / Finish / Cancel builders; runner feedback on successful EscrowCreate.
- Out (deferred): crypto-condition Escrow variant (requires `Condition`/`Fulfillment` fields); time-based lifecycle assertion (ensure Finish respects FinishAfter); Escrow reserve accounting in InvariantPoolBalance (currently flags Escrow-locked XRP as "disappeared"); offer-sequence reuse for OfferCancel (separate milestone).

#### Step 2: Implement tracker

Create `sidecar/internal/fuzz/generator/tracker.go`:

```go
package generator

import (
	mathrand "math/rand/v2"
	"sort"
)

// Tracker holds per-family runtime state so the generator can emit tx types
// that require referencing existing ledger objects (e.g. EscrowFinish needs
// an open escrow). The runner feeds the tracker on successful submits; the
// generator consults it when building.
type Tracker struct {
	escrows *EscrowTracker
}

// NewTracker returns a Tracker with every sub-tracker initialized.
func NewTracker() *Tracker {
	return &Tracker{escrows: newEscrowTracker()}
}

// Escrows returns the escrow sub-tracker.
func (t *Tracker) Escrows() *EscrowTracker { return t.escrows }

// EscrowRef identifies one escrow ledger object by its creator account +
// the Sequence the creating tx was assigned.
type EscrowRef struct {
	Owner    string
	Sequence uint32
}

// EscrowTracker records escrow creations so Finish/Cancel can reference them.
// It never removes records — even after Finish or Cancel, the sequence is
// still valid to reference (rippled returns tecNO_TARGET, which is a useful
// fuzz signal). Production-hardening would prune on observed Finish/Cancel.
type EscrowTracker struct {
	refs []EscrowRef
}

func newEscrowTracker() *EscrowTracker {
	return &EscrowTracker{refs: []EscrowRef{}}
}

// Record adds a new escrow reference. Callers must ensure Owner + Sequence
// correspond to an on-ledger EscrowCreate they have already submitted.
func (e *EscrowTracker) Record(owner string, sequence uint32) {
	e.refs = append(e.refs, EscrowRef{Owner: owner, Sequence: sequence})
}

// Count returns the number of recorded escrows.
func (e *EscrowTracker) Count() int {
	return len(e.refs)
}

// PickOpen returns a deterministic random escrow reference, or ok=false if
// none are tracked. The record list is sorted before indexing so the choice
// is reproducible from the RNG seed.
func (e *EscrowTracker) PickOpen(r *mathrand.Rand) (owner string, sequence uint32, ok bool) {
	if len(e.refs) == 0 {
		return "", 0, false
	}
	// Make a stable copy sorted by (Owner, Sequence).
	snap := make([]EscrowRef, len(e.refs))
	copy(snap, e.refs)
	sort.Slice(snap, func(i, j int) bool {
		if snap[i].Owner != snap[j].Owner {
			return snap[i].Owner < snap[j].Owner
		}
		return snap[i].Sequence < snap[j].Sequence
	})
	pick := snap[r.IntN(len(snap))]
	return pick.Owner, pick.Sequence, true
}
```

```go
package generator

import (
	mathrand "math/rand/v2"
	"strconv"
)

// EscrowCreate generates a simple time-based escrow from one pool account to
// another for 1..10 XRP. FinishAfter is set to "now + 1 second" in XRPL time
// (Ripple epoch starts 2000-01-01, XRPL timestamps are seconds since that).
// This keeps the escrow immediately finishable in the fuzz window while
// staying well-formed (FinishAfter > CloseTime).
func (g *Generator) EscrowCreate(r *mathrand.Rand) (*Tx, error) {
	src, dst := g.pool.PickTwoDistinct(r)
	drops := uint64(r.IntN(10)+1) * 1_000_000

	// Ripple epoch offset: seconds between 1970-01-01 and 2000-01-01 = 946684800.
	// Use a deterministic "now" derived from the RNG — we don't want clock
	// drift to make runs non-reproducible. A large constant is fine; rippled
	// validates FinishAfter > parent-ledger close_time.
	const rippleEpochBias = 946684800
	finishAfter := uint32(rippleEpochBias + 100000 + r.IntN(1000))

	return &Tx{
		Fields: map[string]any{
			"TransactionType": "EscrowCreate",
			"Account":         src.ClassicAddress,
			"Destination":     dst.ClassicAddress,
			"Amount":          strconv.FormatUint(drops, 10),
			"FinishAfter":     finishAfter,
		},
		Secret: src.Seed,
	}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "EscrowCreate",
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.EscrowCreate(r) },
	})
}
```

**Note on RequiresAll:** Escrow is permanently enabled in modern rippled (the amendment landed in 2017 and has been hard-enabled for years). If the smoke turns up `temDISABLED`, add `RequiresAll: []string{"Escrow"}` here — but default to not listing it.

```go
package generator

import (
	"fmt"
	mathrand "math/rand/v2"
)

// EscrowFinish releases a tracked escrow. Without a Condition on the escrow,
// any pool account can finish it after FinishAfter elapses — we pick a random
// pool account as the submitter. Errors if the tracker holds no escrows.
func (g *Generator) EscrowFinish(r *mathrand.Rand) (*Tx, error) {
	owner, seq, ok := g.tracker.Escrows().PickOpen(r)
	if !ok {
		return nil, fmt.Errorf("no tracked escrows to finish")
	}
	submitter := g.pool.Pick(r)
	return &Tx{
		Fields: map[string]any{
			"TransactionType": "EscrowFinish",
			"Account":         submitter.ClassicAddress,
			"Owner":           owner,
			"OfferSequence":   seq,
		},
		Secret: submitter.Seed,
	}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "EscrowFinish",
		CanBuild:        func(g *Generator) bool { return g.tracker.Escrows().Count() > 0 },
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.EscrowFinish(r) },
	})
}
```

```go
package generator

import (
	"fmt"
	mathrand "math/rand/v2"
)

// EscrowCancel cancels a tracked escrow. Submitter is the escrow owner —
// only the owner (or any account if CancelAfter elapsed) can cancel. We use
// the owner for well-formedness.
func (g *Generator) EscrowCancel(r *mathrand.Rand) (*Tx, error) {
	owner, seq, ok := g.tracker.Escrows().PickOpen(r)
	if !ok {
		return nil, fmt.Errorf("no tracked escrows to cancel")
	}
	// Find the wallet for the owner address so we can sign with their seed.
	var ownerSeed string
	for _, w := range g.pool.All() {
		if w.ClassicAddress == owner {
			ownerSeed = w.Seed
			break
		}
	}
	if ownerSeed == "" {
		// Owner not in our pool — fall back to a pool account (will produce
		// tecNO_PERMISSION, which is a legitimate fuzz signal).
		submitter := g.pool.Pick(r)
		ownerSeed = submitter.Seed
	}
	return &Tx{
		Fields: map[string]any{
			"TransactionType": "EscrowCancel",
			"Account":         owner,
			"Owner":           owner,
			"OfferSequence":   seq,
		},
		Secret: ownerSeed,
	}, nil
}

func init() {
	Register(CandidateTx{
		TransactionType: "EscrowCancel",
		CanBuild:        func(g *Generator) bool { return g.tracker.Escrows().Count() > 0 },
		Build:           func(g *Generator, r anyRand) (*Tx, error) { return g.EscrowCancel(r) },
	})
}
```

- **Escrow reserve accounting.** `InvariantPoolBalance` currently considers escrow-locked XRP as "disappeared" — will falsely trigger a monotone-increase violation when an escrow Finishes and the locked amount returns. Track locked amounts in the invariant.
- **Tracker pruning.** Remove refs after observed Finish/Cancel rather than leaving them to produce `tecNO_TARGET` forever.
- **OfferCancel reuse.** Apply the same tracker pattern to OfferCreate so OfferCancel can reference real sequences.
- **AMM family next.** Much more complex — requires AMMCreate with two trust lines, reserve knowledge, and the full LP-token lifecycle. Its own milestone.
